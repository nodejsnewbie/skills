# 技能介绍

此技能用于帮助教师依据《Java高级编程》的双向细目表和复习大纲，从教学 PPT 中提取知识点，并自动化生成**A卷和B卷两套试卷**系统（包括期末试卷、答题卡、参考答案），附带出具**命题双向细目表（Excel格式）**以及**考试试卷审核表**。最终所有的输出文件将被统一组织到一个**新建的独立输出目录**中。

# 目录结构与参考资源

参考模板均存放在本技能的 `resources/` 目录下（需预先拷贝好所有 `.doc`和 `.xls`原件）：

- **大纲与细目表模板**：`Java高级编程复习大纲.docx`、`副本广州华立学院命题双向细目表+java高级编程-A卷-林原.xls`，以及 `副本...-B卷...xls`
- **A卷系列模板**：`1 试卷纸-A卷.doc`、`2 答题纸-A卷.doc`、`3 参考答案及评分标准-A卷.doc`
- **B卷系列模板**：对于B卷同样需要参照B卷的原版模板输出。
- **审核表**：`考试试卷审核表 - 发学院.doc`

# 执行流程

## 步骤 1：新建输出目录并明确出卷要求

1. 在运行此技能前/运行中，首先创建一个**全新的独立目录**（统一以 `output_exams_` 为前缀命名，例如 `output_exams_20231024/`）。**注意：** 根目录下的 `.gitignore` 已配置为忽略所有以 `output_exams_` 开头的文件夹，确保生成的隐私试题文件不会被错误提交到 Git 仓库。
2. 试卷总分为 **100分**，必须包含：
   - **选择题**：30分（15题，每题2分）
   - **填空题**：10分（5空，每空2分）
   - **判断题**：10分（10题，每题1分）
   - **简答题**：30分
   - **综合题**：20分
3. **出卷宗旨**：这是用来直接下发的一份**严肃的真实试卷**，严禁在试卷正文中夹带任何非考试内容的提示、解析或者像“不需要掌握”一类的废话说明。

## 步骤 2：生成 A、B 两套试卷 (Word格式)

基于课件 PPT，紧扣教学大纲（覆盖认知、理解、应用、分析、综合等层次）：

1. 提取题库，拆分编排为难度对等但题目不重合的 **A、B两套试卷**。
2. 依照上述的分值规定进行题目构建。
3. **排版格式极度重要**：在生成两套试卷及其答题纸、参考答案的 Word 文件时，必须严格完全保留给定原版 `.doc` 模板中的格式要素。例如必须**保留左侧装订线（装订线）**设计、大标题样式、表头和计分格。

该步需生成6个Word文件存入输出目录：

- A套：`1 试卷纸-A卷.docx`，`2 答题纸-A卷.docx`，`3 参考答案及评分标准-A卷.docx`
- B套：`1 试卷纸-B卷.docx`，`2 答题纸-B卷.docx`，`3 参考答案及评分标准-B卷.docx`

## 步骤 3：生成试卷审核表 (Word格式)

- 根据试卷的基本信息（课程名：Java高级编程、考试班级、出卷人等），填入审核表内容，生成 `考试试卷审核表.docx`。

## 步骤 4：校验并生成命题双向细目表 (Excel格式)

- 根据 A 卷与 B 卷实际生成的题目进行拆分统计（统计各题是在哪一章节，对应的是认知、理解、应用、分析还是综合层面，以及总共有多少分）。
- 绝不能输出格式随意改变的 MarkDown，必须输出为真实的电子表格 **(.xls 或 .xlsx)**，表格的样式行、列排布（包含各题型的具体填表说明行）必须原模原样遵循 `resources/` 目录下的 `命题双向细目表` 原始文件格式。

# 技能执行指引

请执行该技能的主体脚本以产出结果。请始终确保：

1. 一定生成 **A、B 两套独立、不同的试题试卷体系**；
2. 一定拥有完整的**双向细目表（严格的 Excel 格式）**和**试卷审核表**；
3. 一定是将共计（3+3+1+1=8份或者9份）的真实考务文件干净利落地输出在一个**新创建的专用文件夹**里。
4. 作为一份真实试卷，试卷中杜绝无意义的解析或解释，且严格继承了装订线等固定模板配置。
